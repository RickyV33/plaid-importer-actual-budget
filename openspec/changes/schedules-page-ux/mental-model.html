schedules-page-ux

Change delta — Schedules page UX aligned with the Profiles pattern. Presentation only.

UIeta viewsi18n no backend change

No scheduler, recurrence, persistence, or endpoint-contract changes. Pure presentation.

What changed

  • Creation moved off the list. The always-present "new schedule" form card is gone from /schedules. Creation now lives on its own page at GET /schedules/new (schedules_new.eta), reached from a header "New schedule" button — mirroring the Profiles /profiles/new model. Both the new and edit pages reuse partials/schedules_form; the create form still posts to the unchanged POST /schedules.
  • Header. The list page now opens with a .profiles-header (title + description + button). With zero connections, the button is replaced by the existing "link a connection first" guidance.
  • Visual status. Each non-legacy row shows a status badge — green .badge-active ("Active") or grey .badge-paused ("Paused") — instead of inline "enabled ·/disabled ·" text. Disabled rows get .list-row.is-disabled (dimmed, muted left border; restored to full opacity on hover/focus).
  • Icon toggle. The text Enable/Disable button became an .icon-btn showing fa-toggle-on (enabled) / fa-toggle-off (disabled), labelled via aria-label/title, alongside the edit/delete row icons. Still posts to the unchanged POST /schedules/:id/toggle.

Touched files

  • src/routes/schedules.ts — added GET /schedules/new.
  • src/views/schedules.eta — header, badges, dimmed rows, icon toggle.
  • src/views/schedules_new.eta — new dedicated creation page.
  • public/style.css — .badge-active, .badge-paused, .list-row.is-disabled.
  • src/i18n/en.ts, src/i18n/es.ts — schedules.statusActive / schedules.statusPaused.
