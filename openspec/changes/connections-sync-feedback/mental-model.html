Change delta — per-connection sync results that persist until dismissed, no auto-reload.

UIeta + client JS/sync response i18nno migration

Additive response fields; no change to rate-limit math or how transactions are fetched/pushed.

What changed

  • Per-connection breakdown in /sync. The response now returns synced: [{ itemId, imported, lastSyncedAt }] (built in routes/sync.ts from syncAccountResults.importedByItemForRun(runId) defaulting each attempted connection to 0) and adds itemId to every skipped entry. status, totalImported, and runId are unchanged.
  • Sticky, dismissible messages. Each connection card has a .conn-messages slot; a synced connection shows "Imported N transaction(s)", a throttled one shows "Skipped (sync limit reached): retry in ~M min". The overall total renders below the sync buttons. Every message has an × close control (buildMsg) and the whole set auto-clears after 60s as a fallback.
  • No auto-reload. The old setTimeout(reload, 1800) is gone. Instead each card's .conn-last-sync line is updated in place from synced[].lastSyncedAt, so timing stays current without discarding the messages.
  • Client state only. Dismissals are transient DOM state — a rate-limit skip goes stale as its window clears, so nothing is persisted server-side (unlike the durable banners).
