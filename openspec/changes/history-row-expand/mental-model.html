history-row-expand

Change delta — the History run row is the drill-down toggle, only when it imported something.

UIeta viewno backend change

Presentation only; same recorded data and columns.

What changed

  • Toggle moved onto the row. The separate "per-account results" <details> summary row is gone. The run's own summary row is now the disclosure control (caret, role="button", tabindex, aria-expanded; click or Enter/Space), revealing a hidden .history-detail row with the same account/status/imported/reason table.
  • Only when imported > 0. The drill-down is offered only for runs with totalImported > 0. Runs that imported nothing — including the 0-import marker-only no-op pulls from rate-limit-count-all-pulls — render as plain, non-expandable rows, keeping the list calm and hiding marker noise.
  • Trade-off. A failed run (0 imported) is no longer expandable; its failure status still shows in the status column.

Touched files

  • src/views/history.eta — row-as-toggle, totalImported > 0 gate, hidden detail row, toggle script.
  • public/style.css — .history-row.is-expandable + caret rotation.
