What changed

  • Flat nav. Connections and Profiles are now top-level pages (GET /connections, GET /profiles) beside Schedules, History, and Settings. The in-page tab buttons, .tab-panel switching, and the home-tab sessionStorage script are gone — the URL carries the state.
  • View split. home.eta → connections.eta (link / sync / relink / manage-accounts JS) + profiles.eta (mapping / pending / delete JS). The panel markup and handlers move verbatim.
  • Leaner loads. The Connections page no longer makes the per-profile listAccountsForProfile Actual Budget calls that home.ts:76 ran on every visit; only the Profiles page pays for them.
  • Dashboard at /. A new read-only landing page with four linked status cards — connections (+ relink alert), profiles, last sync, next scheduled sync. Derived entirely from local DB (plaidItems, profiles, schedules.next_run_at); it makes no Plaid/Actual calls and exposes no create/edit/delete controls. Each card has a calm empty state.
  • Admin-only card. For admins, one extra card shows the number of other registered users (users.count() − 1), hidden from members. The user store has no activity/last-login tracking, so "active" is read as total registered — no new query, no migration; a sole admin sees an empty state, not "0".

i18n

  • nav.home removed; nav.connections + nav.profiles added.
  • New dashboard.* namespace (en + es): card labels, relink alert, empty states.
  • Existing home.* content keys move verbatim with the panel markup — not renamed (kept to avoid mass churn; a possible future tidy-up).

Out of scope

  • No change to connection/profile/mapping/sync/schedule behavior — only where those surfaces live.
  • No DB migration, no endpoint-contract change, no new dependency.
  • The dashboard adds zero external I/O; any action belongs on the page a card links to.
