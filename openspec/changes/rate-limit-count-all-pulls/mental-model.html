No change to pull/drain/import logic, limit config, or skip behavior.

What changed

  • The bug. The limiter counts pulls via runs that wrote to sync_account_results, but those rows are written during DRAIN. A pull that imports nothing (empty delta) or hits a connection mapped to no profile drains nothing, records no row, and so was never counted — the ceiling never tripped no matter how many times you synced.
  • The fix. After drain + failure recording, runSync records a 0-import marker row (reason="pulled", profile_id NULL, a representative targeted account) for every successfully pulled connection that produced no result row this run. The selection is the pure helper pulledItemsNeedingMarker(targets, errored, withResults).
  • No double counting. countPullsForItemSince counts DISTINCT runs, and a marker is only added when a connection has no other row this run, so connections that drained real results are untouched. importedByItemForRun sums the marker as 0, so dashboard / connections per-connection totals are unaffected.
