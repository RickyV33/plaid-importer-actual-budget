schedule-naming-and-layout

Change delta — schedules gain an optional custom name and a cleaner, multi-line row.

UIeta viewsi18n 1 migrationadditive column

One nullable column; no scheduler, recurrence, fan-out, or endpoint-contract changes.

What changed

  • Optional name. Schedules gain a nullable name column (migration 0012_schedule_name.sql). The shared partials/schedules_form.eta has a name input used by both create and edit; schedules.create/update trim it and store blank as NULL (normalizeName in queries.ts).
  • Name fallback. The list title is the name when set, otherwise the joined connection names — so pre-existing (unnamed) schedules look exactly as before. Names are optional and non-unique.
  • Three-line row. A non-legacy row now shows the title + status badge on one line (.schedule-title-row), the recurrence (days · time · repeat) on .schedule-recurrence, and the connections + next-run on .schedule-meta, instead of one crammed muted line.
  • 12-hour time. The read view formats time_of_day as h:mm AM/PM via a template-local fmt12 helper; the stored value and the edit form's <input type="time"> stay 24-hour. Recurrence text also now uses the existing schedules.at/repeatEvery/repeatWeeksUnit keys (previously hardcoded English).
  • Legacy rows unchanged. Legacy schedules (days_of_week IS NULL) keep their single-line interval rendering and warning.
