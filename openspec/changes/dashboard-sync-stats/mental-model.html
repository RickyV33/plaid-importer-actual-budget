Read-only; aggregates existing sync_runs. No new persistence, no Plaid/Actual calls.

What changed

  • Per-connection sync status. The dashboard previously collapsed timing into one global "last sync" (max over connections) and one "next sync" (min over schedules). It now shows a .dash-sync table with each connection's own last_synced_at and its next scheduled sync — the soonest next_run_at among the owner's enabled schedules whose plaid_item_ids include that connection (computed in home.ts via a connection→soonest map). A connection with no enabling schedule shows "No schedule".
  • Windowed import totals. A new card sums sync_runs.total_imported over rolling 7/30/60/90-day windows (syncRuns.totalImportedSince(owner, cutoff), returns 0 when none). Windows are independent cumulative sums (30d includes 7d). Card links to /history.
  • Still read-only. Only new read queries; no create/edit/delete controls and no external network calls enter the dashboard path. Connection-count and profile-count cards are unchanged; the History and Schedules links are preserved as footer links under the sync table.
