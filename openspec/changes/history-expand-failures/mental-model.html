Change delta — failed runs are expandable again so their per-account reasons are visible.

Refines history-row-expand; presentation only.

What changed

  • Failures expand. The History drill-down gate widened from totalImported > 0 to results.length > 0 && (totalImported > 0 || status === "failure"). A failed run imports nothing but its per-account rows carry the reason, which is what you need to diagnose it.
  • No-op pulls stay plain. A successful run that imported nothing (marker-only) is still non-expandable, preserving the calm list from history-row-expand.

Touched files

  • src/views/history.eta — widened the expandable condition.
